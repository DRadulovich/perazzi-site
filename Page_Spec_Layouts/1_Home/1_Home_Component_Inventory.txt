1. HeroBanner
Used in: Act I – Hero (“Invitation”)
Role: Full-bleed, full-viewport hero section.
Purpose: Establishes the first impression — conveys prestige, calm authority, and emotional tone of the brand.
Content: Background image or looping video, overlaid headline (brand mantra) and optional subheading.
Behavior:
Fade-in on page load (ease-out, 700 ms).
Parallax drift on scroll (disabled if prefers-reduced-motion).
Static until scroll moves to next section.
Implementation notes:
Must preload hero media (LCP target ≤ 2.5 s).
Uses <h1> heading and Tailwind theme tokens for text color.
Uses Next/Image for responsive optimization.

2. ScrollIndicator
Used in: Act I – Hero (“Invitation”)
Role: Visual cue prompting scroll interaction.
Purpose: Non-essential decorative icon (often an SVG arrow or text “scroll ↓”) indicating more content below.
Behavior:
Gentle bounce or pulse animation loop (CSS-based).
Fixed near bottom of viewport; fades as scroll begins.
Accessibility:
aria-hidden="true" if decorative; or focusable if implemented as an interactive button.
Must meet contrast standards if text-based.

3. TimelineScroller
Used in: Act II – Craftsmanship Journey (“Immersion in Craft”)
Role: Scroll-driven container that orchestrates the storytelling sequence.
Purpose: Presents the stages of bespoke gun-making as an immersive narrative.
Behavior:
On desktop: pinned scroll with overlay transitions.
On mobile: collapses into stacked sections.
Registers scroll-depth analytics for each stage.
Contains: Multiple TimelineItem components.
Implementation notes:
Uses Intersection Observer + Framer Motion to trigger reveals.
Each stage fade/slide animation: opacity 0→1, Y +20 px→0, duration 600 ms.
Provides skip-link for keyboard navigation.

4. TimelineItem
Used within: TimelineScroller (multiple instances).
Role: Represents a single stage of the build process (e.g., Measurement & Fit, Engraving, Proof).
Purpose: Combines explanatory text with authentic factory imagery or video.
Content: Stage title, descriptive paragraph, and a linked FactoryAsset.
Behavior:
Fades and slides in when entering viewport midpoint.
Supports fallback static layout if JS disabled.
Accessibility:
Text and media remain in DOM order; captions describe visuals.
High-contrast overlays ensure readability.

5. MarqueeFeature
Used in: Act III – Champion Legacy (“Belonging”)
Role: Feature banner spotlighting a champion or legacy figure.
Purpose: Connects emotional narrative of ownership and achievement.
Content:
Portrait or action image.
Quote or testimonial.
Optional “Read Full Story” link referencing an Article in CMS.
Behavior:
Subtle fade/slide-in animation; quote highlight on hover/tap.
No autoplay motion or video loops.
Accessibility:
Quote marked up with <blockquote> and <cite> when applicable.
Text separate from imagery (never flattened).

6. CTASection
Used in: Act IV – Finale (“Join the Legacy”)
Role: Closing call-to-action banner.
Purpose: Converts emotional resonance into user action.
Content: Closing statement and a single primary CTA button (e.g. “Begin Your Journey”).
Behavior:
Fades up as section enters viewport.
CTA button animates subtly on hover/focus; prefetches target route.
Implementation notes:
Background: solid or textured dark theme.
Lightweight, minimal scripts; instant response to click.
Tailwind focus ring or Radix UI Button for a11y.

7. Global Defaults
Header / Footer: Persist globally but excluded from this page spec.
Framework: Next.js (App Router + React 18).
Styling: Tailwind CSS + Radix UI primitives.
Animation: Framer Motion for timeline and section reveals.
Data Sources (Sanity CMS):
Champion, FittingStage, FactoryAsset, Article schemas.
Performance Targets: LCP ≤ 2.5 s, CLS ≤ 0.05.
Accessibility: WCAG AA contrast, full keyboard navigation, motion reduction support.

(Global site header and footer are also present on the page by default, but they lie outside the scope of this home page content spec.)
